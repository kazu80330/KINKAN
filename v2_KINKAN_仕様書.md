## 1. プロジェクト背景と開発方針

### なぜこのアプリを作るのか

業務管理を個人レベルで行うための勤怠管理ツールが必要だった。既存のSaaSツールは多機能すぎたり月額コストがかかったりするため、**自分専用のシンプルなローカルアプリ**として開発することにした。

### 開発方針

- **ローカルファイル1枚で動く**ことを最優先とする。サーバーもフレームワークも不要。
- **一般公開しない**個人専用ツールであるため、デザインや機能は自分の使い方に最適化する。
- データは**Google Sheetsで管理**することで、PCを変えてもデータを引き継げるようにする。
- 金銭的コストは極力ゼロに近づける。

### コストに関する経緯

当初はローカルのExcelファイルとの連携を実装した。しかし「複数のPCで同じデータを使いたい」という要件が生まれたため、Google Sheetsへの切り替えを決断した。

Google Sheetsを選んだ理由：
- Google Drive 15GB まで無料
- Google Sheets API は1日300リクエストまで無料（このアプリの用途では超えない）
- Google Cloud Consoleへのカード登録は必要だが、無料枠内であれば課金されない
- HTMLファイル自体をどのPCに置いても、同じGoogleアカウントでログインすれば同じデータにアクセスできる

## 2. 技術選定の理由

### なぜHTMLシングルファイルか

サーバーもデプロイも不要で、ファイルをダブルクリックするだけで使える手軽さを重視した。ただし`file://`プロトコルではGoogle APIが動作しないため、Pythonのローカルサーバー（`python -m http.server 8080`）を経由して`http://localhost:8080`で開く運用とした。

### なぜChromeか

Google Identity Services（OAuth 2.0）およびIndexedDB（ファイルハンドルの保存）との相性が最も良く、動作が安定しているため。

### なぜPython 3.12.xか

- バージョン3.13は2024年10月リリースでライブラリ対応が不十分な場合がある
- 3.12は2028年10月まで長期サポートが続く
- 今後データ分析やAI（pandas、PyTorch等）の学習に進む可能性を考慮すると、ライブラリ互換性が最も充実している

### 使用ライブラリ

| ライブラリ | 用途 | 選定理由 |
|---|---|---|
| xlsx.js 0.18.5 | ローカルxlsxのダウンロード | CDNから読み込むだけで使える |
| Google Sheets API v4 | データのクラウド同期 | 無料・安定・公式 |
| Google Identity Services | OAuth 2.0認証 | Googleの公式ライブラリ |
| Noto Sans JP / Space Mono | フォント | 日本語対応・視認性 |

## 4. 機能仕様

### 4.1 Google Sheets連携バナー

#### 背景・こだわり

当初はローカルExcelファイルとの連携だったが、複数PCでのデータ共有が必要になりGoogle Sheetsへ切り替えた。**ChromeのFile System Access APIがローカルファイル（`file://`）では動作しない**問題に苦労した経緯があり、Google APIのポーリング方式での読み込みを採用している。

#### 仕様

| 項目 | 内容 |
|---|---|
| ログイン | Google Identity Services OAuth 2.0。スコープは `spreadsheets` と `drive.file` |
| 新規作成 | ログイン後、Google Drive内に「KINKAN 勤怠記録」スプレッドシートを自動作成 |
| 自動同期 | 記録変更後2秒のデバウンスで自動書き込み |
| ログアウト | トークン失効 + STATEリセット + UI全初期化（出退勤・休憩・ポモドーロの表示をすべてリセット） |
| スプレッドシートリンク | 連携中は「📋 開く」リンクを表示してスプレッドシートに直接アクセスできる |
| API準備完了の通知 | Google APIの読み込み完了時にボタンのテキストを「🔑 Googleログイン（準備完了）」に変更する |

#### 複数アカウント対応

別のGoogleアカウントを使う場合は、Google Cloud ConsoleのOAuth同意画面でテストユーザーとして追加するだけでよい。アカウントごとに別々のスプレッドシートが作成されるため、データは独立して管理される。

---

### 4.2 出退勤管理

#### 背景・こだわり

単純な出退勤記録だけでなく、**同日に複数回出退勤できる**ことが重要な要件だった。理由は「夜中に作業して日付を超えてしまう場合」を考慮したためで、日跨ぎ勤務を正確に記録できるようにしている。

また、労働基準法上の休憩と「再出勤時の離席時間」は別物であるため、厳密に区別して管理することにこだわった。

#### レイアウト

出退勤カードは**3列×2行のグリッド**で表示する。

```
┌──────────┬──────────┬──────────┐
│ 出勤時刻  │ 勤務時間  │ 休憩時間  │
├──────────┼──────────┼──────────┤
│ 退勤時刻  │総勤務時間 │累計休憩時間│
└──────────┴──────────┴──────────┘
        [ ▶ 出勤 ]  [ ■ 退勤 ]
```

| 表示項目 | 内容 |
|---|---|
| 出勤時刻 | 現セッションの出勤時刻（再出勤時は再出勤時刻にリセット） |
| 退勤時刻 | 最後の退勤時刻（日跨ぎは25:00:xx形式） |
| 勤務時間 | 現セッションのみの勤務時間（再出勤時は0からカウント） |
| 総勤務時間 | 全セッションの累計勤務時間 |
| 休憩時間 | 現在進行中の1回分の休憩時間 |
| 累計休憩時間 | 確定済み + 進行中の休憩合計 |

#### 出勤ボタンの動作

**初回出勤時：**
- STATEを全リセット
- 新規レコードを作成・保存
- ポモドーロ自動スタート設定がONの場合はタイマーを自動開始

**同日再出勤時（複数回出勤）：**
- 退勤〜再出勤の間の「離席時間」は`awaySec`として別途記録する
- **休憩時間には加算しない**（労基法上の休憩と区別するため）
- 出勤時刻は再出勤した時刻にリセット
- 勤務時間のカウントは0からリスタート
- 総勤務時間には前セッションの分が引き継がれる

#### 退勤ボタンの動作

- **日跨ぎ対応：** 出勤日の0時を基準に経過時間を計算。翌1時退勤なら`25:00:xx`形式で表示。日跨ぎの場合は`(+1)`も付記する。
- 退勤後も同日中であれば出勤ボタンが再度押せる（複数回出勤に対応）
- 退勤後のステータスは「離席中（再出勤できます）」と表示

#### 時刻・時間の表記形式

すべての時刻・時間は**hh:mm:ss形式**で統一する。ただしポモドーロタイマーのみ**mm:ss形式**とする（作業・休憩の残り時間を直感的に把握するため）。

### 4.3 ポモドーロタイマー

#### 背景・こだわり

業務中の集中管理のためにポモドーロ法を採用。25分作業・5分休憩のサイクルを自動で繰り返すことで、手動で管理する手間をなくしたい。

**ポモドーロ休憩中の時間は勤務時間に含めない**ことが重要なこだわり。休憩は休憩として明確に区別し、実質的な作業時間のみを勤務時間として記録する。同様に、タイマーを一時停止している間も「作業していない時間」であるため、休憩時間として扱う。

#### 仕様

| 項目 | 内容 |
|---|---|
| 作業時間 | 25分 |
| 休憩時間 | 5分 |
| 自動連続 | 作業完了→休憩タイマーを即座に自動スタート。休憩完了→作業タイマーを即座に自動スタート |
| 一時停止 | 一時停止中の時間は累計休憩時間に加算する（勤務時間から除外） |
| ポモドーロ休憩中 | 休憩中の時間は勤務時間から除外し、累計休憩時間に加算する |
| スキップ | 作業スキップ→休憩へ移行。休憩スキップ→作業へ移行。各経過時間を適切に確定する |
| リセット | 現在のモードをリセット。一時停止中の場合は経過分を休憩に確定する |
| 自動スタート設定 | 出勤時に自動スタートするかをチェックボックスで設定。**デフォルトON**。設定はlocalStorageに保存 |
| ステータス表示 | 作業中：「🍅 ポモドーロ作業中」 / 休憩中：「🍅 ポモドーロ休憩中（勤務時間外）」 |
| 完了音 | Web Audio APIを使ったビープ音（作業完了・休憩完了で異なる音） |

---

### 4.4 休憩管理

#### 背景・こだわり

労働基準法上の休憩（昼休みなど）を正確に記録するためのセクション。ポモドーロの5分休憩とは別に、任意のタイミングで長い休憩を取ることができる。

**休憩中は勤務時間のカウントを停止する**ことが必須要件。当初の実装では休憩中も勤務時間が加算されていたため修正した。

#### 休憩管理カードの下部レイアウト

```
[ 休憩開始時刻 ] [ 休憩時間 ] [ 累計休憩時間 ] [ 休憩回数 ]
```

| 項目 | 内容 |
|---|---|
| 休憩開始時刻 | 現在の休憩を開始した時刻（hh:mm:ss） |
| 休憩時間 | 現在進行中の1回分の休憩経過時間（リアルタイム更新） |
| 累計休憩時間 | 確定済み休憩 + 現在進行中の合計（リアルタイム更新） |
| 休憩回数 | 手動休憩ボタンを押した回数のみカウント |

#### 休憩開始ボタンの動作

- ポモドーロタイマーを停止し、**作業モード（25:00）にリセット**する
- ポモドーロ休憩中だった場合はその経過時間を確定して累計に加算する
- 勤務時間のカウントを停止する

#### 休憩終了ボタンの動作

- 休憩時間を確定してtotalBreakSecondsに加算する
- **ポモドーロ作業タイマーを自動スタート**する（休憩終了後は集中モードへ即移行）

### 4.5 勤務記録（当月）

#### 背景・こだわり

当月の記録を一覧で確認できるセクション。複数回出勤した日はすべてのセッションを確認できるようにしたい。しかし常に展開表示していると見づらいため、**アコーディオン形式**（クリックで展開）を採用した。

日付は**昇順（1日が上）**で表示する。月の流れに沿って記録を確認できる方が自然であるため。

#### カラム構成

```
[ 日付 ] [ 出勤時刻 ] [ 退勤時刻 ] [ 勤務時間 ] [ 総勤務時間 ] [ 休憩時間 ]
```

#### 表示ルール

- **1回のみ出勤の日：** 1行で表示
- **複数回出勤の日：** 折りたたみ表示。クリックすると各セッションの出勤・退勤・勤務時間を展開表示。ヘッダー行に総勤務時間と休憩時間を表示
- **勤務中の記録：** 退勤欄に「勤務中」と緑色で表示
- **30秒ごとに自動保存**（勤務中にブラウザを閉じてもデータが残るよう）

### 4.6 月次レポートページ

#### 背景・こだわり

月単位での勤務状況を把握するためのページ。**後から記録を手動で修正・追加できる**ことが重要な要件。手動で追加・修正した記録は、自動記録と区別できるよう視覚的に明示する。

ページの順序は「勤務記録テーブル → 棒グラフ」とした。まずデータを確認し、グラフで全体の傾向を把握するという自然な流れのため。

#### サマリーカード

| 項目 | 計算方法 |
|---|---|
| 出勤日数 | 退勤済みレコードの件数 |
| 総勤務時間 | 全レコードのworkSec合計（hh:mm:ss） |
| 平均勤務時間 | 総勤務時間 ÷ 出勤日数 |
| 🍅 総ポモドーロ | 完了したポモドーロセッション数の合計 |

#### 月次勤務記録テーブル

- その月の**全日付を表示**（未勤務日は空白のまま）
- 土日は日付を赤色表示
- 今日の行は紫色でハイライト
- **行をクリックすると編集モーダルが開く**（未勤務日は新規追加モーダル）

#### 手動編集の視覚的区別

後から手動で追加・編集した記録は以下で識別できる：
- テーブルの日付欄に黄色の「手動」バッジを表示
- 棒グラフのバーが**黄色**になる（通常は紫、今日は緑）
- 再度編集モーダルを開くと「手動編集済み（日時）」の警告を表示

これは「後から追加したことが分かるように工夫してほしい」という要件から実装した。

#### 棒グラフ

- 日別の勤務時間を棒グラフで表示
- **y軸は1時間ごとの目盛り**とグリッドラインを表示（最大勤務時間に合わせて自動スケール）
- 「⬇ PNG保存」ボタンでCanvasに描画したグラフをPNG画像としてダウンロード可能
- ファイル名：`KINKAN_YYYY年M月_勤務時間.png`

#### 月ナビゲーション

‹ › ボタンで過去・未来の月に移動できる。記録がない月は空白テーブルが表示される。

#### 編集モーダル

| 編集項目 | 入力形式 |
|---|---|
| 出勤時刻 | hh:mm:ss（秒単位） |
| 退勤時刻 | hh:mm:ss（秒単位） |
| 休憩回数 | 数値 |
| ポモドーロ数 | 数値 |

- 時刻から勤務時間を自動計算（日跨ぎも考慮）
- 保存時に`manual: true`と`manualAt`（保存日時）を記録
- 削除ボタンでそのレコードを削除
- ESCキーまたはオーバーレイクリックで閉じる

## 5. データ仕様

### 5.1 localStorageスキーマ

**キー：** `kinkan_records`

構造：`{ "YYYY-MM": [ record, ... ] }` の月別オブジェクト

#### recordオブジェクト

| フィールド | 型 | 内容 |
|---|---|---|
| `date` | string | `YYYY/MM/DD` |
| `clockIn` | string | 出勤時刻（`hh:mm:ss`） |
| `clockInRaw` | string | ISO 8601（ページリロード後の復元用） |
| `clockOut` | string \| null | 退勤時刻（日跨ぎは`25:xx:xx`形式） |
| `clockOutRaw` | string | ISO 8601 |
| `workSec` | number | 総勤務時間（秒） |
| `breaks` | number | 手動休憩回数 |
| `totalBreakSec` | number | 累計休憩時間（秒） |
| `awaySec` | number | 再出勤時の離席時間（秒、休憩とは別管理） |
| `pomodoros` | number | 完了ポモドーロ数 |
| `sessions` | array | `[{in: ISO, out: ISO}]` 各セッションの出退勤時刻 |
| `manual` | boolean | 手動編集フラグ |
| `manualAt` | string | 手動編集日時（ISO 8601） |
| `overnight` | boolean | 日跨ぎフラグ |

#### その他のlocalStorageキー

| キー | 内容 |
|---|---|
| `kinkan_autoPomodoro` | ポモドーロ自動スタート設定（`"1"` or `"0"`） |
| `kinkan_gs_token` | Google OAuthアクセストークン |
| `kinkan_spreadsheet_id` | Google SheetsのスプレッドシートID |

### 5.2 Google Sheetsスキーマ

- スプレッドシート名：**「KINKAN 勤怠記録」**（年は含めない。年をまたいでも1つのスプレッドシートで管理するため）
- 月ごとにシートを分割（例：「2026年04月」）
- **全日付を行として出力**（未勤務日も空白行で表示。月のカレンダーとして完全な形を保つため）
- 同期のたびに月末行の下に棒グラフを自動挿入（既存グラフは削除して再作成）

#### 列定義

| 列 | ヘッダー | 内容 |
|---|---|---|
| A | 日付 | `YYYY/MM/DD` |
| B | 曜日 | 日〜土 |
| C | 出勤時刻 | `hh:mm:ss` |
| D | 退勤時刻 | `hh:mm:ss`（日跨ぎは`25:xx:xx`） |
| E | 勤務時間 | `hh:mm:ss` |
| F | 休憩回数 | 数値 |
| G | 休憩合計 | `hh:mm:ss` |
| H | ポモドーロ数 | 数値 |
| I | 手動編集 | `TRUE` / `FALSE` |

## 6. 状態管理（STATEオブジェクト）

アプリの実行時状態はすべてグローバルな`STATE`オブジェクトで管理する。ページリロード時は`initToday()`でlocalStorageから復元する。

| フィールド | 型 | 説明 |
|---|---|---|
| `isWorking` | boolean | 勤務中フラグ |
| `clockInTime` | Date | 現セッションの出勤時刻 |
| `clockOutTime` | Date | 最後の退勤時刻 |
| `firstClockInTime` | Date | その日最初の出勤時刻 |
| `isOnBreak` | boolean | 手動休憩中フラグ |
| `breakStartTime` | Date | 手動休憩開始時刻 |
| `breakCount` | number | 手動休憩回数 |
| `totalBreakSeconds` | number | 確定済み累計休憩時間（秒） |
| `totalAwaySeconds` | number | 再出勤の離席時間累計（秒） |
| `sessions` | array | 完了セッション履歴 |
| `sessionWorkSec` | number | 完了セッションの勤務時間合計（秒） |
| `pomodoroMode` | string | `"work"` or `"break"` |
| `pomodoroRunning` | boolean | タイマー動作中フラグ |
| `pomodoroSeconds` | number | 残り秒数 |
| `pomodoroTotal` | number | 設定秒数（1500 or 300） |
| `pomodoroCount` | number | 完了ポモドーロ数 |
| `pomodoroBreaking` | boolean | ポモドーロ休憩中フラグ |
| `pomodoroBreakStart` | Date | ポモドーロ休憩開始時刻 |
| `pomodoroPauseStart` | Date | 一時停止開始時刻 |
| `pomodoroPausedSec` | number | 一時停止累計秒数 |

## 7. 勤務時間計算ロジック

### 計算式

```
現セッション勤務時間 = (現在時刻 − 現セッション出勤時刻)
                      − 手動休憩中経過時間
                      − ポモドーロ休憩中経過時間
                      − 一時停止中経過時間

総勤務時間 = 完了セッション合計(sessionWorkSec)
           + 現セッション経過時間
           − totalBreakSeconds（確定済み累計休憩）
           − 進行中の休憩時間

累計休憩時間 = totalBreakSeconds
             + 手動休憩中経過時間
             + ポモドーロ休憩中経過時間
             + 一時停止中経過時間
```

### 休憩時間の確定タイミング

以下のタイミングで進行中の休憩時間が`totalBreakSeconds`に確定される：

1. 手動休憩終了ボタン押下時
2. ポモドーロ休憩タイマー完了時
3. ポモドーロスキップ（休憩→作業）時
4. ポモドーロリセット時（休憩中だった場合）
5. 退勤ボタン押下時（進行中の休憩をすべて確定）
6. ポモドーロ一時停止解除時（一時停止時間を確定）
7. 手動休憩開始時（ポモドーロ休憩中だった場合はその分を確定）

### 離席時間の扱い

再出勤時（複数回出勤）の退勤〜再出勤の間の時間は`awaySec`として記録されるが、**`totalBreakSeconds`には加算しない**。これは労働基準法上の休憩と、業務の合間の離席（例：一時帰宅、外出など）を区別するためのこだわりによるもの。

## 8. 起動環境とファイル構成

### ファイル構成

```
任意のフォルダ/
├── kinkan.html     # アプリ本体（HTML/CSS/JS 単一ファイル）
├── KINKAN.bat      # ワンクリック起動用バッチファイル
└── README.txt      # 使い方ガイド
```

> **重要：** 3ファイルは必ず同じフォルダに置くこと。`KINKAN.bat`は自身のフォルダをサーバーのルートとして起動するため、別フォルダにあると動作しない。

### KINKAN.batの動作

1. port 8080を使用中のプロセスを終了
2. `python -m http.server 8080` をバックグラウンドで起動
3. 2秒待機
4. `http://localhost:8080/kinkan.html` をChromeで自動オープン
5. 「続行するには何かキーを押してください...」で待機（このウィンドウを閉じるとサーバーが停止）

### 動作要件

| 要件 | 内容 |
|---|---|
| ブラウザ | Google Chrome（最新版） |
| Python | 3.12.x以上 |
| インターネット接続 | Google Fonts・外部ライブラリ・Google APIの読み込みに必要 |
| Googleアカウント | Google Sheets連携に必要 |

OAuth同意画面はテストモード（未審査）のため、ログイン時に「このアプリは確認されていません」という警告が表示される場合がある。「詳細」→「KINKANに移動」で続行できる。個人専用ツールのため審査申請は行わない。

## 10. マルチPC運用

### 同一アカウントでの複数PC運用

1. `kinkan.html`・`KINKAN.bat`・`README.txt` の3ファイルをコピー
2. コピー先のPCにPythonをインストール（3.12.x推奨）
3. `KINKAN.bat` をダブルクリックして起動
4. 同じGoogleアカウントでログインすれば同じスプレッドシートにアクセスできる

> **注意：** `localStorage`はPC・ブラウザ固有のため、別PCではlocalStorageのデータは引き継がれない。ただしGoogle Sheetsのデータは共有されるため、ログイン後にデータが同期される。

### 別アカウントでの運用

別のGoogleアカウントでKINKANを使いたい場合：
1. Google Cloud ConsoleのOAuth同意画面でそのGmailアドレスをテストユーザーに追加
2. そのアカウントでログインすると「＋ 新規作成」が表示されるので、そのアカウント専用のスプレッドシートが作成される
3. データはアカウントごとに独立して管理される

## 11. データリセット手順

テストデータを消して本番運用を開始する場合：

| 手順 | 操作 |
|---|---|
| 1 | `kinkan.html`を開いた状態でF12 → Applicationタブ → Local Storage → `http://localhost:8080` を右クリック → Clear |
| 2 | [drive.google.com](https://drive.google.com) で「KINKAN 勤怠記録」をゴミ箱に移動 → ゴミ箱を空にする |
| 3 | `KINKAN.bat` を再起動 → Googleログイン → 「＋ 新規作成」をクリック |

## 12. 既知の制限事項

| 制限 | 詳細 |
|---|---|
| ブラウザ依存 | Google ChromeのみでGoogle Sheets連携が動作する |
| localhost必須 | `file://`プロトコルではGoogle APIが動作しないため、Pythonサーバー経由が必須 |
| トークン有効期限 | OAuthトークンには有効期限があり、期限切れ時は再ログインが必要 |
| localStorage非共有 | localStorageはPC・ブラウザ固有のためPC間で共有されない |
| OAuth未審査 | テストモードのため「確認されていません」警告が表示される場合がある |
| グラフ再作成 | Google Sheetsへの同期のたびにグラフが削除・再作成される |
| 複数アカウント同時ログイン | 1つのブラウザウィンドウで同時に複数アカウントを使用することはできない |
